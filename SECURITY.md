# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly. Do **not** open a public GitHub issue for security-related bugs.

Instead, please send an email ```oveldman@gmail.com``` or contact the maintainer directly through GitHub. Include as much detail as possible so the issue can be assessed and addressed quickly.

## Supported Versions

Only the latest version on the `main` branch is actively maintained and receives security fixes.

## Disclosure

Once a vulnerability is confirmed and fixed, a summary will be shared publicly. We appreciate your patience and responsible disclosure.
